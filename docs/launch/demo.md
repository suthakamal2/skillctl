# skillctl Terminal Demo Script

This script outlines a ~30-second terminal recording to demonstrate skillctl.
Please capture this using `asciinema` and convert it to a gif via `agg` (for a smaller file size), OR capture it directly with `terminalizer`.

## Setup
Ensure you are in an empty directory. Have a sample rule file ready to demonstrate, or rely on the sample generated by `skillctl init`.

## Script

**[0:00 - 0:03]**
Command: `skillctl init`
Expected: Instantly creates `.skillctl/rules/`, `bundles.yaml`, and a sample rule.
Timing: 3 seconds.

**[0:03 - 0:08]**
Command: `cat .skillctl/rules/safe-deploy.md`
Expected: Shows a markdown file with YAML frontmatter containing `id`, `tier`, `triggers`, and `deps`.
Timing: 5 seconds.

**[0:08 - 0:11]**
Command: `skillctl build`
Expected: Instantly scans the rules directory and emits `index.yaml` with token totals.
Timing: 3 seconds.

**[0:11 - 0:18]**
Command: `skillctl suggest "we need to deploy v2 to staging tonight"`
Expected: Rich output showing suggested bundles (e.g., `deploy (overlap 3/3)`) and suggested rules in tier order.
Timing: 7 seconds.

**[0:18 - 0:24]**
Command: `skillctl why deploy`
Expected: Rich output showing exactly which rules matched (e.g., `safe-deploy (tier 2)`) and why, along with any near misses.
Timing: 6 seconds.

**[0:24 - 0:28]**
Command: `skillctl bundle deploy`
Expected: Dumps the concatenated markdown content of the `deploy` bundle, demonstrating transitive dependency resolution within token budgets.
Timing: 4 seconds.

**[0:28 - 0:32]**
End-card display.
Text to overlay or echo: "skillctl 0.1 — github.com/suthakamal2/skillctl — MIT"
Timing: 4 seconds.
