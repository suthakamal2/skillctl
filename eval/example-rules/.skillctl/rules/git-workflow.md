# Git Workflow

Short-lived branches, rebase to keep history linear, small reviewable PRs.

## When this applies
Loads when the prompt mentions: branch, merge, rebase, pull request.

## Guidance
- Short-lived branches, rebase to keep history linear, small reviewable PRs.
- Make the safe choice the default; document the exception when you deviate.
- Prefer the boring, well-understood option over the clever one.
